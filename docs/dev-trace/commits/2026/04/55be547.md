## Descripción
Este cambio registra el commit `fix(kb): persist training-kb.json to GCS on Cloud Run` dentro del sistema de trazabilidad del proyecto. Se modificaron 2 archivos: .env.example, server/lib/trainingKB.js.

Contexto del commit:
Cloud Run is stateless — every deploy discards local file writes.
All autolearned KB entries were lost on each deployment.

trainingKB.js:
- USE_GCS=true when K_SERVICE env (Cloud Run) + GCS bucket configured
- loadTrainingKB(): GCS-backed with 60s in-memory cache (avoids GCS
  read on every chat turn); falls back to local file if GCS unavailable
- saveTrainingKB(): writes local snapshot + async GCS upload; cache
  updated immediately so next read sees new data without round-trip
- Startup init: pre-loads GCS → local mirror on import in Cloud Run
- getTrainingPaths(): now includes gcs path for diagnostics

Default bucket: GCS_QUOTES_BUCKET (bmc-cotizaciones, already exists)
Default object: kb/training-kb.json (private, no allUsers needed)
Override: GCS_KB_BUCKET / GCS_KB_OBJECT env vars
