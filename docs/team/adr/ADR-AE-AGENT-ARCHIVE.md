# ADR — AE-Agent quote archive location

**Date:** 2026-05-06
**Status:** Accepted
**Context:** PR #110 — Panelin agent as native calculator tool surface (28 tools, end-to-end quoting through the agent)

## Decision

Quotes generated by the AE-Agent (Panelin chat path or the MCP `/api/agent/exec-tool` path) **share the same archive** as human-driven quotes: the existing GCS-backed [`quoteRegistry`](../../server/lib/quoteRegistry.js) at `gs://${GCS_QUOTES_BUCKET}/registry/{pdf_id}.json`.

A `source: "ae_agent" | "calculator"` field on each registry entry distinguishes the two lineages. Default is `"calculator"`; the agent path passes `"ae_agent"` explicitly through `POST /calc/cotizar/pdf`.

## Alternatives considered

**B1 — Postgres `ae_quotes` table.** Better for cross-tenant analytics + JOINs against ML/Shopify. **Deferred** because it requires a new migration and the agent doesn't currently need JOINs that GCS can't answer with `listJsonInGcs` + filter.

**B2 — Sheets archive via `POST /api/cotizaciones`.** Human-visible in `Master_Cotizaciones`. **Deferred** because `guardar_en_crm` already writes the GCS link to `CRM_Operativo` col AH (`LINK_PRESUPUESTO`), covering the visibility need without a second write path. Sales-ops can read both views.

**B3 — Extend the existing GCS-backed registry.** **Chosen.** Lowest delta, single source of truth for the agent's recall surface (`obtener_cotizacion_por_id`, `listar_cotizaciones_recientes`, `cancelar_cotizacion`, `historial_cliente` all read this store today). Tabular audit available without migration.

## Consequences

- **Single recall surface:** filtering "show me agent-generated quotes today" is a thin filter over the existing `listJsonInGcs` listing. `listar_cotizaciones_recientes` accepts an optional `source: "ae_agent"` param to surface this in the agent UX.
- **Provenance preserved:** every agent-generated quote carries `source: "ae_agent"` from the point of generation onward. Backward compatible — older quotes default to `"calculator"`.
- **Structured log trail:** `generar_pdf` emits a `console.log({event: "ae_agent_quote", scenario, lista, duration_ms, ok, pdf_id})` per call. Cloud Logging / pino scrapers can filter on `event="ae_agent_quote"` for an out-of-process audit independent of the registry. PII-free.
- **No backfill:** existing pre-PR-#110 entries have no `source` field and read as `"calculator"` via the default. Acceptable — historical quotes were all human-driven.

## Future paths (out of scope here)

- **Postgres B1** if cross-tenant analytics arrive (e.g. JOINs against ML/Shopify orders).
- **Sheets B2** if sales-ops asks for an in-Sheets agent-quote view distinct from the existing CRM_Operativo col AH link.
- **60s idempotency window** (`requestHash` dedup): low real-world hit rate today, deferrable.
- **Suggest-response audit** (`server/routes/bmcDashboard.js`): grep for LLM-only price emitters; gate any survivors through `/calc/cotizar`. Independent task.
