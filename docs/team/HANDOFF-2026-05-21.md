## Three corrections from advisor before applying the Phase 1 migration

1. **RLS policies as drafted were inert** — the BMC backend connects via `pg` pool with a privileged Postgres role that bypasses RLS. The `current_setting('request.jwt.claims')` calls would return NULL because PostgREST is never in the path. Real per-user enforcement is the `WHERE actor_user_id = $req.user.id` clause in handler code (matches `/api/me/notifications`, `/api/me/quotes` convention). Solution: kept `ALTER TABLE … ENABLE ROW LEVEL SECURITY` as defense-in-depth (matches other `identity.*` tables) but dropped the placeholder policies; documented in table comment.
2. **`event_id uuid` → `BIGSERIAL`** — every other audit table in the repo (`audit_log.audit_id`, `quote_events.event_id`, `tk_audit_log.audit_id`) uses BIGSERIAL. UUIDs cost 2× bytes per row + per index entry, don't sort temporally. At 10M rows/year that adds up.
3. **Client-emit needed an allowlist** — `ACTION_TAXONOMY` validation only gates the action *name*; a signed-in user could POST `{"action":"quote.complete"}` and it would land with `client_emitted=true`. Added `CLIENT_EMITTABLE` subset (`nav.*`, `ui.*`, `auth.session.end`) — server returns 403 `action_not_client_emittable` for anything outside it.
